Logic Explanation

Quick logic chart to show what happens in each step of the application.

The loot array The lootItems array is used as a single source of truth. I placed each item in the array and use it in each updateUI function It contained all items with name/quantity/price and allows to manage the items in one place. By manage I mean add/remove. This approach allows to quickly calculate totals and render the list of items.

Why quantity was added In Phase 1 assessment I added only name and price. While in this one we added quantity. With this new parameter calculation are made a bit different.

Where updateUI() performs The updateUI() function is the function that is used everywhere and after each user action. This allows to control calculations and renders in one place and not repeat code in other places.

How state controls The state allows to manage visibility of some compoennts and affect them thru functions or logic. In our case with states were able to show/hide messages.

Where conditional logic prevents invalid state Every user action goes through validation before any calculations happen. With this approach it is possible to avoid fails and invalid states. Also, it prevens devition to 0 or other invalid cases.
